So what about ES6 Javascript?
  • Overall, good support http://kangax.github.io/compat-table/es6
  • Transpilers for syntax
  • Shims for objects and functions
  • chrome://flags to enable "experimental Javascript features"

Variable improvements:
  • let and const variables are all block-scoped
  • declared where needed
  • constants clarify their purpose
  • easier to read, small optimisations

For-of improvements:
  • works with break, continue and return - unlike forEach()
  • can loop over data and strings - unlike for-in
  • more concise, also works with set and map objects

Sets:

  • ordered list of values without duplicates
  • compare values, rather than accessing
  • adding to set is chainable
  • delete individuals or clear all

See the Pen ES6 Sets by Gem Barrett (@gembarrett) on CodePen.

Maps:

  • in ES5, maps were regular objects and keys were stringified
  • in ES6, keys and values can be any type
  • setters and getters store and retrieve data
  • has(), clear() and delete() available (as with sets)
  • use .size to determine number of items in map
